# Youbestar Agent 开发交接文档

## 项目定位

这是一个本地可控的 AI Agent 原型系统。

当前目标不是做复杂平台，而是先完成最小闭环：

- 网页对话
- 模型配置
- 第三方 API 模型列表发现
- 闲聊模式 / 任务优先模式
- Agent Action 解析
- 技能调用
- 技能审批
- 技能注册表
- GitHub 同步上传

后续可以继续扩展 Excel、OCR、ERP、浏览器自动化和社区技能安装系统。

## 主要模块说明

### `server.py`

负责：

- 返回 `index.html`
- 读写模型配置
- 接收聊天请求
- 调用 Agent loop
- 挂载技能系统 API

主要接口：

```text
GET  /health
GET  /
GET  /config
POST /config
POST /models/discover
POST /chat
```

`POST /chat` 请求体支持：

```json
{
  "message": "你好",
  "allowChat": true,
  "history": []
}
```

`allowChat=True`：允许自然回复，模型可以输出 `Response:`。

`allowChat=False`：任务优先，只允许 `Thought / Action / Params`，后端会丢弃自然回复。

### 模型列表发现

配置页填写 API 地址和 API Key 后，可以调用：

```text
POST /models/discover
```

请求：

```json
{
  "api_url": "https://api.deepseek.com/v1",
  "api_key": "..."
}
```

后端会把兼容地址转换为 `/models`，携带 Bearer Key 请求第三方接口，并统一提取模型 ID。响应不会返回 API Key：

```json
{
  "models_url": "https://api.deepseek.com/v1/models",
  "models": ["deepseek-chat", "deepseek-reasoner"]
}
```

支持 OpenAI-compatible 的 `data: [{"id": "..."}]` 和常见 `models` 列表格式。不支持模型发现的第三方接口仍可手动填写模型名。

当用户只填写供应商根地址，且根 `/models` 返回 `404` 时，后端只会再尝试一次 `/v1/models`。成功后响应中的 `api_url` 会返回实际 `/v1` 基地址，前端同步更新并自动保存，避免聊天请求继续使用错误路径。

### `core/llm.py`

负责请求 OpenAI-compatible 云端模型。

API 地址从 `youbestar.json` 读取。

大部分云端模型只要是 `/v1/chat/completions` 兼容格式，都可以接入。

### `core/loop.py`

Agent 主循环。

核心逻辑：

1. 构造 prompt
2. 要求模型输出 `Thought / Action / Params`
3. 根据 `allowChat` 决定是否允许 `Response`
4. 解析模型输出
5. 判断 Action 是否注册、是否开启
6. 调用技能
7. 返回模型输出、工具结果和可选用户回复

当前 prompt 已要求模型使用命名空间 Action：

```text
official.open_browser
local.xxx
community.author.xxx
```

### `core/parser.py`

负责解析模型输出：

```text
Thought: ...
Action: official.open_browser
Params: {"url": "https://www.baidu.com"}
Response: 你好
```

返回：

- `thought`
- `action`
- `params`
- `response`

`Response` 是可选字段，只在 Chat On 场景用于自然回复。

### `agent_system/manager.py`

技能系统核心。

负责：

- 创建目录
- 校验命名空间
- 写入 sandbox
- 写入测试
- 运行测试
- 提交审批
- 批准/拒绝
- 注册技能
- 加载并运行技能

关键设计：

```text
skills/
  official/
  community/
  local/
  registry.json
```

### `agent_system/skill_registry.py`

负责：

- 技能开关
- 技能卡片列表
- 兼容旧 Action 名称

例如旧名称：

```text
open_browser
```

会映射为：

```text
official.open_browser
```

## 技能命名规则

必须使用命名空间：

```text
official.skill_name
community.author.skill_name
local.skill_name
```

含义：

- `official`：官方维护技能，可随项目版本升级
- `community`：社区共享技能，可下载、导入、升级
- `local`：本地用户技能，不自动更新

模型新写技能时，只允许进入：

```text
local.skill_name
```

## 技能注册表

文件：

```text
agent_system/skills/registry.json
```

示例：

```json
{
  "official.open_browser": {
    "path": "agent_system/skills/official/open_browser.py",
    "version": "1.0.0",
    "source": "official",
    "title": "打开浏览器",
    "description": "根据 URL 打开系统默认浏览器。",
    "author": "youbestar"
  }
}
```

运行逻辑：

```text
Action -> registry.json -> skill path -> import module -> run(params)
```

## 安全边界

模型允许：

- 写入 `agent_system/sandbox/`
- 写入测试
- 运行 sandbox 测试
- 提交审批
- 读取白名单内普通项目文件

模型禁止：

- 直接修改 `skills/`
- 删除文件
- 操作系统文件
- 读取密钥、token、cookie、auth 文件
- 调用 ERP、微信、发布、删除等高风险动作
- 执行未批准代码

安全扫描会拦截部分高风险 import 和函数调用。

## GitHub 交接

GitHub 仓库：

```text
https://github.com/bori830421-pixel/youbestar
```

两个 BAT 是相互独立的单向操作：

- `github_upload.bat`：以本地版本为准，上传并覆盖 GitHub。
- `github_sync.bat`：以 GitHub 为来源，只下载同步。

上传脚本会：

- 初始化 `.git`
- 设置远端 origin
- 设置分支为 `main`
- 检查 `youbestar.json` 是否被跟踪
- add / commit
- 刷新远端分支元数据，但不会 pull
- 使用 `git push --force-with-lease` 安全覆盖远端

下载同步脚本会：

- 初始化 `.git`
- 设置远端 origin
- 检查未提交的本地修改；发现修改立即停止并保护文件
- 检查远端 main 是否存在
- fetch
- 使用 `pull --ff-only` 下载；历史分叉时停止，不自动改写历史
- 永远不会 push

空仓库第一次没有远端 `main` 分支，需要先运行上传脚本。

- 根因：部分 OpenAI 兼容中转接口返回 UTF-8 JSON，但响应头没有正确声明字符集，`requests.Response.json()` 会产生中文乱码。

## 2026-06-05 项目规则固化：技能输出与联网层

- 新增根目录规则文件：`AGENTS.md`
- 固化原则 1：技能只负责业务执行并返回结构化数据，不负责最终 Markdown 渲染；最终用户可见回复统一走 `core/ui_formatter.py`。
- 固化原则 2：联网技能不各自处理编码、请求头、超时和 JSON 解析；后续应新增或复用 `core/http_client.py`，统一提供 `fetch_text()`、`fetch_json()`。
- 编码策略：共享网络层负责从响应头识别 charset，并自动尝试 `utf-8-sig`、`utf-8`、`gb18030`、`gbk`，避免每个技能手写“某接口用 GBK”。
- 固化原则 3：技能按能力域归类，不无限平铺增长。建议域包括 `query_hub`、`market_data`、`weather_data`、`web_search`、`browser_headless`、`browser_desktop`。
- 浏览器能力拆分：桌面可见浏览器用于打开页面、登录、人工查看；无头浏览器用于后台搜索、抓取、提取和校验。
- 本次为文档/规则固化，未改业务代码。

## 2026-06-05 证券行情表格与字段兜底修复

- `synthesize_node`：对搜索结果做证据综合。
- `answer_check_node`：让 LLM 检查最终回答是否覆盖必答点，漏答时追加补充说明。
- `finalize_node`：统一进入 `core.ui_formatter.py`，保持用户可见回复格式。

- 临时记忆不会进入模型上下文。
- 长期记忆按业务模块隔离。
- Agent Prompt 只包含短期历史和已确认长期业务记忆，不包含未确认候选。
