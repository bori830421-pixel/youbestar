# Youbestar Agent 开发交接文档

## 项目定位

这是一个本地可控的 AI Agent 原型系统。

当前目标不是做复杂平台，而是先完成最小闭环：

- 网页对话
- 模型配置
- 第三方 API 模型列表发现
- 闲聊模式 / 任务优先模式
- LangGraph 旁路实验模式
- Agent Action 解析
- 技能调用
- 技能审批
- 技能注册表
- GitHub 同步上传

后续可以继续扩展 Excel、OCR、ERP、浏览器自动化和社区技能安装系统。

## 主要模块说明

### `server.py`

负责：

- 返回 `index.html`
- 读写模型配置
- 接收聊天请求
- 调用 Agent loop
- 挂载技能系统 API

主要接口：

```text
GET  /health
GET  /
GET  /config
POST /config
POST /models/discover
POST /chat
POST /langgraph/chat
```

`POST /chat` 请求体支持：

```json
{
  "message": "你好",
  "allowChat": true,
  "history": []
}
```

`allowChat=True`：允许自然回复，模型可以输出 `Response:`。

`allowChat=False`：任务优先，只允许 `Thought / Action / Params`，后端会丢弃自然回复。

### 模型列表发现

配置页填写 API 地址和 API Key 后，可以调用：

```text
POST /models/discover
```

请求：

```json
{
  "api_url": "https://integrate.api.nvidia.com/v1",
  "api_key": "..."
}
```

后端会把兼容地址转换为 `/models`，携带 Bearer Key 请求第三方接口，并统一提取模型 ID。响应不会返回 API Key：

```json
{
  "models_url": "https://integrate.api.nvidia.com/v1/models",
  "models": ["model-a", "model-b"]
}
```

支持 OpenAI-compatible 的 `data: [{"id": "..."}]` 和常见 `models` 列表格式。不支持模型发现的第三方接口仍可手动填写模型名。

当用户只填写供应商根地址，且根 `/models` 返回 `404` 时，后端只会再尝试一次 `/v1/models`。成功后响应中的 `api_url` 会返回实际 `/v1` 基地址，前端同步更新并自动保存，避免聊天请求继续使用错误路径。

`POST /langgraph/chat` 是独立实验入口。请求额外传入：

```json
{
  "message": "打开百度",
  "allowChat": false,
  "threadId": "conversation-id"
}
```

响应会额外返回：

```json
{
  "engine": "langgraph",
  "thread_id": "conversation-id",
  "graph_nodes": ["plan", "execute_skill", "reflect", "finish"],
  "turn_count": 1
}
```

### `core/langgraph_bridge.py`

最小 LangGraph 旁路桥接。

状态图流程：

```text
plan -> no_action / execute_skill -> reflect -> finish
```

它复用现有 Prompt、解析器、技能审批状态和技能开关。图使用 `InMemorySaver` 按 `threadId` 隔离对话历史与轮次状态，仅用于能力测试；服务重启后状态会清空。
`reflect` 节点会在技能执行或无操作后做规则复盘：保留已有自然回复、把成功工具结果桥接为回复、把失败工具结果转成自然解释、为空的闲聊回复提供兜底。

### `core/llm.py`

负责请求 OpenAI-compatible 云端模型。

API 地址从 `youbestar.json` 读取。

大部分云端模型只要是 `/v1/chat/completions` 兼容格式，都可以接入。

### `core/loop.py`

Agent 主循环。

核心逻辑：

1. 构造 prompt
2. 要求模型输出 `Thought / Action / Params`
3. 根据 `allowChat` 决定是否允许 `Response`
4. 解析模型输出
5. 判断 Action 是否注册、是否开启
6. 调用技能
7. 返回模型输出、工具结果和可选用户回复

当前 prompt 已要求模型使用命名空间 Action：

```text
official.open_browser
local.xxx
community.author.xxx
```

### `core/parser.py`

负责解析模型输出：

```text
Thought: ...
Action: official.open_browser
Params: {"url": "https://www.baidu.com"}
Response: 你好
```

返回：

- `thought`
- `action`
- `params`
- `response`

`Response` 是可选字段，只在 Chat On 场景用于自然回复。

### `agent_system/manager.py`

技能系统核心。

负责：

- 创建目录
- 校验命名空间
- 写入 sandbox
- 写入测试
- 运行测试
- 提交审批
- 批准/拒绝
- 注册技能
- 加载并运行技能

关键设计：

```text
skills/
  official/
  community/
  local/
  registry.json
```

### `agent_system/skill_registry.py`

负责：

- 技能开关
- 技能卡片列表
- 兼容旧 Action 名称

例如旧名称：

```text
open_browser
```

会映射为：

```text
official.open_browser
```

## 技能命名规则

必须使用命名空间：

```text
official.skill_name
community.author.skill_name
local.skill_name
```

含义：

- `official`：官方维护技能，可随项目版本升级
- `community`：社区共享技能，可下载、导入、升级
- `local`：本地用户技能，不自动更新

模型新写技能时，只允许进入：

```text
local.skill_name
```

## 技能注册表

文件：

```text
agent_system/skills/registry.json
```

示例：

```json
{
  "official.open_browser": {
    "path": "agent_system/skills/official/open_browser.py",
    "version": "1.0.0",
    "source": "official",
    "title": "打开浏览器",
    "description": "根据 URL 打开系统默认浏览器。",
    "author": "youbestar"
  }
}
```

运行逻辑：

```text
Action -> registry.json -> skill path -> import module -> run(params)
```

## 安全边界

模型允许：

- 写入 `agent_system/sandbox/`
- 写入测试
- 运行 sandbox 测试
- 提交审批
- 读取白名单内普通项目文件

模型禁止：

- 直接修改 `skills/`
- 删除文件
- 操作系统文件
- 读取密钥、token、cookie、auth 文件
- 调用 ERP、微信、发布、删除等高风险动作
- 执行未批准代码

安全扫描会拦截部分高风险 import 和函数调用。

## GitHub 交接

GitHub 仓库：

```text
https://github.com/bori830421-pixel/youbestar
```

两个 BAT 是相互独立的单向操作：

- `github_upload.bat`：以本地版本为准，上传并覆盖 GitHub。
- `github_sync.bat`：以 GitHub 为来源，只下载同步。

上传脚本会：

- 初始化 `.git`
- 设置远端 origin
- 设置分支为 `main`
- 检查 `youbestar.json` 是否被跟踪
- add / commit
- 刷新远端分支元数据，但不会 pull
- 使用 `git push --force-with-lease` 安全覆盖远端

下载同步脚本会：

- 初始化 `.git`
- 设置远端 origin
- 检查未提交的本地修改；发现修改立即停止并保护文件
- 检查远端 main 是否存在
- fetch
- 使用 `pull --ff-only` 下载；历史分叉时停止，不自动改写历史
- 永远不会 push

空仓库第一次没有远端 `main` 分支，需要先运行上传脚本。

## 2026-06-04 中文乱码修复

- 根因：部分 OpenAI 兼容中转接口返回 UTF-8 JSON，但响应头没有正确声明字符集，`requests.Response.json()` 会产生中文乱码。
- 修复：`core/llm.py` 直接从响应原始字节按 `utf-8-sig` 解析 JSON。
- 回归测试：`tests/test_llm.py`
- 当前完整测试结果：24 项通过。

## 2026-06-04 对话记录持久化修复

- 根因：前端 `conversations` 只保存在页面内存中，关闭网页或刷新后会丢失。
- 修复：`index.html` 使用 `localStorage` 保存对话列表、当前选中对话和消息历史；重新进入应用时自动恢复。
- 细节：保存时过滤 `pending` 临时消息，避免把“正在思考...”状态持久化。
- 附带修复：`agent_system/manager.py` 的 `to_relative()` 兼容 Windows 临时目录长路径/短路径名不一致，避免完整测试在注册技能用例中误报。
- 回归测试：`tests/test_conversation_persistence_ui.py`
- 当前完整测试结果：26 项通过。

## 2026-06-04 工具结果回复桥接修复

- 根因：Agent 执行技能后只返回 `action_result`，如果模型没有提前写 `Response`，用户回复区域不会自然展示工具结果。
- 修复：新增 `core.loop.bridge_tool_result_to_response()`，在 `allowChat=True` 且存在工具 Action 时，把工具执行结果桥接为 `response`。
- 覆盖范围：普通 `/chat` Agent loop 和 `core/langgraph_bridge.py` LangGraph 旁路都已接入。
- 效果：查询天气这类技能调用会直接在用户回复区域返回天气结果，不再只显示在工具卡片里。
- 回归测试：`tests/test_agent_loop.py`、`tests/test_langgraph_bridge.py`
- 当前完整测试结果：33 项通过。

## 2026-06-04 自然聊天展示修复

- 根因：后端 `reply` 和前端主气泡都偏调试日志风格，会把 `模型输出 / 工具执行结果 / Thought / Action / Params` 暴露给用户，闲聊显得冷冰冰。
- 修复：`server.py` 的 `reply` 改为用户可见最终答案；调试信息继续保留在 `model_reply`、`thought`、`action`、`params`、`action_result` 字段。
- 前端：`index.html` 默认关闭思考和工具过程展示，主气泡优先显示 `response` / `reply` / `content`，调试按钮打开后仍可查看 Thought、Action、Params、Result 和 LangGraph 节点。
- Prompt：`core/loop.py` 明确要求 Response 是直接给用户看的自然、简洁、温暖回复，不要提内部协议字段。
- 回归测试：`tests/test_chat_reply.py`、`tests/test_natural_chat_ui.py`、`tests/test_agent_loop.py`
- 当前完整测试结果：40 项通过。

## 2026-06-04 LangGraph 复盘节点

- 根因：原 LangGraph 旁路仍是单次计划、单次执行、直接结束，缺少执行后的观察和复盘。
- 修复：`core/langgraph_bridge.py` 增加 `reflect` 节点，流程升级为 `plan -> no_action / execute_skill -> reflect -> finish`。
- 复盘规则：已有自然回复则保留；工具成功但没有回复时把结果桥接为回复；工具失败时把 `未知工具 / 技能已关闭 / 失败 / 错误 / error` 转成自然解释；无工具且无回复时给出温和兜底。
- 效果：`/langgraph/chat` 比普通 `/chat` 多一层“执行 -> 观察 -> 复盘 -> 回答”的思考结构。
- 回归测试：`tests/test_langgraph_bridge.py`

## 2026-06-04 Youbestar 自研 Agent Runtime 骨架

- 战略定位：Youbestar 最终要实现自己的独立智能体内核，LangGraph 只作为参考教材和实验对照，不作为正式核心依赖。
- 架构文档：`docs/architecture/youbestar-agent-runtime.md`
- 新增状态层：`core/agent_state.py`
- 新增节点层：`core/agent_nodes.py`
- 新增运行时：`core/agent_runtime.py`
- 新增检查点接口：`core/agent_checkpoint.py`
- 当前最小自研流程：`prepare -> execute -> reflect -> finalize`
- 行为：复用现有 prompt、parser、技能注册表、技能开关和工具执行；执行后通过 reflect 节点复盘结果；finalize 输出兼容现有 chat 响应字段的 `AgentResult`。
- `/chat` 已默认切换到自研 `AgentRuntime`；旧 `agent_loop` 仍作为兼容 fallback 保留。
- 回归测试：`tests/test_agent_runtime.py`
- 当前完整测试结果：45 项通过。
