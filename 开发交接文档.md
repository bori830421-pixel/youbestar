# Youbestar Agent 开发交接文档

## 项目定位

这是一个本地可控的 AI Agent 原型系统。

当前目标不是做复杂平台，而是先完成最小闭环：

- 网页对话
- 模型配置
- 闲聊模式 / 任务优先模式
- Agent Action 解析
- 技能调用
- 技能审批
- 技能注册表
- GitHub 同步上传

后续可以继续扩展 Excel、OCR、ERP、浏览器自动化和社区技能安装系统。

## 主要模块说明

### `server.py`

负责：

- 返回 `index.html`
- 读写模型配置
- 接收聊天请求
- 调用 Agent loop
- 挂载技能系统 API

主要接口：

```text
GET  /health
GET  /
GET  /config
POST /config
POST /chat
```

`POST /chat` 请求体支持：

```json
{
  "message": "你好",
  "allowChat": true,
  "history": []
}
```

`allowChat=True`：允许自然回复，模型可以输出 `Response:`。

`allowChat=False`：任务优先，只允许 `Thought / Action / Params`，后端会丢弃自然回复。

### `core/llm.py`

负责请求 OpenAI-compatible 云端模型。

API 地址从 `youbestar.json` 读取。

大部分云端模型只要是 `/v1/chat/completions` 兼容格式，都可以接入。

### `core/loop.py`

Agent 主循环。

核心逻辑：

1. 构造 prompt
2. 要求模型输出 `Thought / Action / Params`
3. 根据 `allowChat` 决定是否允许 `Response`
4. 解析模型输出
5. 判断 Action 是否注册、是否开启
6. 调用技能
7. 返回模型输出、工具结果和可选用户回复

当前 prompt 已要求模型使用命名空间 Action：

```text
official.open_browser
local.xxx
community.author.xxx
```

### `core/parser.py`

负责解析模型输出：

```text
Thought: ...
Action: official.open_browser
Params: {"url": "https://www.baidu.com"}
Response: 你好
```

返回：

- `thought`
- `action`
- `params`
- `response`

`Response` 是可选字段，只在 Chat On 场景用于自然回复。

### `agent_system/manager.py`

技能系统核心。

负责：

- 创建目录
- 校验命名空间
- 写入 sandbox
- 写入测试
- 运行测试
- 提交审批
- 批准/拒绝
- 注册技能
- 加载并运行技能

关键设计：

```text
skills/
  official/
  community/
  local/
  registry.json
```

### `agent_system/skill_registry.py`

负责：

- 技能开关
- 技能卡片列表
- 兼容旧 Action 名称

例如旧名称：

```text
open_browser
```

会映射为：

```text
official.open_browser
```

### `agent_system/server.py`

技能系统的 FastAPI router。

主要接口：

```text
GET  /skills/prompt
GET  /skills/read-policy
POST /skills/files/list
POST /skills/files/read
GET  /skills/approved
GET  /skills/registry
POST /skills/toggle
GET  /skills/approvals
POST /skills/sandbox/write
POST /skills/tests/write
POST /skills/tests/run
POST /skills/approval/request
POST /skills/approve
POST /skills/reject
```

## 技能命名规则

必须使用命名空间：

```text
official.skill_name
community.author.skill_name
local.skill_name
```

含义：

- `official`：官方维护技能，可随项目版本升级
- `community`：社区共享技能，可下载、导入、升级
- `local`：本地用户技能，不自动更新

模型新写技能时，只允许进入：

```text
local.skill_name
```

## 技能注册表

文件：

```text
agent_system/skills/registry.json
```

示例：

```json
{
  "official.open_browser": {
    "path": "agent_system/skills/official/open_browser.py",
    "version": "1.0.0",
    "source": "official",
    "title": "打开浏览器",
    "description": "根据 URL 打开系统默认浏览器。",
    "author": "youbestar"
  }
}
```

运行逻辑：

```text
Action -> registry.json -> skill path -> import module -> run(params)
```

## 安全边界

模型允许：

- 写入 `agent_system/sandbox/`
- 写入测试
- 运行 sandbox 测试
- 提交审批
- 读取白名单内普通项目文件

模型禁止：

- 直接修改 `skills/`
- 删除文件
- 操作系统文件
- 读取密钥、token、cookie、auth 文件
- 调用 ERP、微信、发布、删除等高风险动作
- 执行未批准代码

安全扫描会拦截部分高风险 import 和函数调用。

## GitHub 交接

GitHub 仓库：

```text
https://github.com/bori830421-pixel/youbestar
```

当前仓库是空仓库时：

1. 先运行 `github_upload.bat`
2. 完成首次 commit 和 push
3. 后续再用 `github_sync.bat` 同步

不要先运行 sync，因为空仓库没有远端 `main` 分支。

上传脚本会自动：

- 初始化 `.git`
- 设置远端 origin
- 设置分支为 `main`
- 检查 `youbestar.json` 是否被跟踪
- add / commit / push

同步脚本会自动：

- 初始化 `.git`
- 设置远端 origin
- 检查远端 main 是否存在
- fetch
- pull --rebase

## 已知注意事项

- `youbestar.json` 是本地配置，不能上传 GitHub。
- 官方技能文件当前是包装模块，实际逻辑仍复用 `tools/`。
- community skill 安装还没有做导入包流程，目前只预留了目录和命名空间。
- local skill 审批通过后会注册为 `local.*`。
- 前端技能页现在只做管理和展示，不做 marketplace。
