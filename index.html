Skills 推荐清单

这是当前 Youbestar Agent 掌握的技能管理页。左侧每一项都是独立技能，可以打开或关闭；关闭后的技能不会出现在 Agent 提示词里，后端执行时也会再次拦截。

技能现在按命名空间注册：official.* 是官方维护技能，community.author.* 是社区共享技能，local.* 是你的本地技能。所有正式可调用技能都会写入 agent_system/skills/registry.json。

维护规范

  • 模型只能写入 sandbox 和测试目录，不能直接修改正式 skills。
  • 所有新技能必须有清晰输入输出，并提供测试用例。
  • 测试通过后只能提交审批请求，批准后进入 skills/local 并自动注册为 local.*。
  • 涉及浏览器、文件读取、外部接口的技能默认保持谨慎，必要时可随时关闭。

安全优先

  • 默认把未审批技能当作不可调用代码。
  • 读取文件仅允许白名单目录内的普通项目文件，密钥、token、auth、cookie、数据库和缓存目录会被拦截。
  • ERP、微信、发布、删除、保存、提交等高风险动作不在当前技能系统中开放。
  • 技能开关是运行时控制项，关闭后即使模型输出对应 Action，也不会执行。

控制智能体是否可以读取和修改自身项目代码。

自我进化权限

开启后，Youbestar Agent 可以在当前项目白名单目录内读取普通代码文件、写入普通项目文件、安装或覆盖本地技能，并为自己的能力迭代准备测试。

关闭后，聊天仍可使用普通工具和技能，但读取项目文件、写入项目文件、写技能、安装本地技能等自修改动作会被后端拦截。

开启后允许

读取普通项目代码文件、写入普通项目文件、创建或覆盖 local.* 技能、写入技能测试，并让运行时按这些能力继续工作。

始终保护

密钥、token、cookie、.env、youbestar.json、.git、虚拟环境、缓存、数据库和本地业务数据目录仍然禁止读取或写入。

执行边界

这个开关会随每次聊天请求发送到后端；后端执行工具前会再次检查，避免只靠界面状态决定权限。

检测到业务信息

是否保存到长期业务记忆？确认后才会参与 ERP、订单、客户、SKU、财务等业务推理。
